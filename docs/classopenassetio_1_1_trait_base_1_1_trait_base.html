Detailed Description

A single-class module that provides the TraitBase class.

A trait view provides a way to hide the underlying dictionary-like data access from hosts and managers. Trait view classes wrap a TraitsData instance and provide member functions that query/mutate properties on the data.

This base class provides the common interface for a concrete trait view.

As an example, assume a trait view called MyTrait and an arbitrary TraitsData instance. Before we can extract MyTrait property values from the data we must check that it supports MyTrait. We can then use the trait's concrete accessors to retrieve data from the underlying dictionary-like data.

A trait class inheriting from this base must have a static kId attribute member giving the unique string ID of that trait.

See also
kTraitId

In addition, the derived class should implement appropriate typed accessor / mutator methods that internally call the wrapped data's getTraitProperty / setTraitProperty.

Note
Attempting to access a trait's properties without first ensuring the data holds that trait via isValid, or otherwise, may trigger an exception if the trait is not set in the data.
